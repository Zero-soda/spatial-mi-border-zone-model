# GitHub and Zenodo Release Checklist

1. Confirm or update the public GitHub repository from the contents of this folder.
2. Confirm remaining co-author and corresponding-author ORCID identifiers if required by the submission system.
3. Verify creator metadata in `.zenodo.json` and `CITATION.cff`.
4. Commit the repository and create the versioned release `v0.5.0`.
5. Create a new Zenodo version, archive the release, and confirm its version DOI.
6. Cite the final version DOI in the release notes and the stable concept DOI
   `10.5281/zenodo.21203188` in reusable manuscript metadata.
7. Confirm that third-party raw GEO data are linked by accession number rather than redistributed.
