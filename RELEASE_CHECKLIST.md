# GitHub and Zenodo Release Checklist

1. Create a public GitHub repository from the contents of this folder.
2. Confirm remaining co-author and corresponding-author ORCID identifiers if required by the submission system.
3. Replace placeholder creator fields in `.zenodo.json` and `CITATION.cff`.
4. Commit the repository and create a versioned release, for example `v0.1.0`.
5. Archive the GitHub release on Zenodo and obtain the version DOI.
6. Replace the manuscript GitHub URL and Zenodo DOI placeholders before journal upload.
7. Confirm that third-party raw GEO data are linked by accession number rather than redistributed.
